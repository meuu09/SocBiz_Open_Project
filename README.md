# Agentic AI-Based Dynamic Tariff Optimization for EV Charging Networks

SocBiz Open Project 2026 — Analytics Vertical | Society of Business, IIT Roorkee

## Overview

EV charging stations today use static, fixed-rate pricing (₹15/kWh) regardless of demand. This leads to congestion at peak hours and empty stations at off-peak hours, wasting infrastructure and losing revenue.

This project builds a multi-agent AI system that dynamically adjusts tariffs in real time based on predicted demand, using two real-world datasets from Caltech (USA) and Shenzhen (China). The system learns over episodes getting better at pricing decisions with each run.

## Architecture

```
UrbanEV Dataset
      │
      ▼
Agent 1: Demand Prediction Agent
      │   XGBoost → predicts charger_util_rate per grid × hour
      │   Output: DemandSignal (predicted utilization per district)
      ▼
Agent 2: Tariff Pricing Agent
      │   Optimal tariff (₹8–₹24/kWh) is selected depending on the different pricing zones
      │   ACN Revenue Model → informs pricing decisions
      │   Output: bandit_tariff + rule_tariff per district-hour
      ▼
Agent 3: Monitor & Learning Agent
      │   Evaluates all 6 metrics using elasticity simulation
      │   Sends weighted reward back → Pricing Agent Q-values update
      └── Feedback loop → Agent 2 learns over episodes
```

Each agent follows a structured **Observe → Think → Act** cycle with persistent long-term memory (JSON) and a decision log (CSV).


## Datasets

| Dataset | Source | Description |
|---|---|---|
| ACN-Data | Caltech/JPL | 14,848 real EV charging sessions (Apr–Dec 2018) from a workplace charger at Caltech |
| UrbanEV / ST-EVCDP | Shenzhen, China | 30 days × 247 districts × 5-minute intervals — occupancy, price, volume, duration |

**ACN** is used for revenue baseline (₹134.14/session at ₹15/kWh), customer response rate, and pricing efficiency score.

**UrbanEV** is used for demand prediction, charger utilization, off-peak uplift, and wait time reduction.

## Metrics Tracked (per judges' specification)

| Metric | Data Source | Description |
|---|---|---|
| Revenue Gain % | ACN baseline | Dynamic vs static ₹15/kWh revenue |
| Charger Utilization Rate | UrbanEV | Before vs elasticity-simulated after pricing |
| Off-Peak Uplift | UrbanEV | Session increase in discount zones |
| Avg Wait Time Reduction | UrbanEV | Queue proxy reduction converted to minutes |
| Customer Response Rate | ACN + elasticity | % session change from price change |
| Pricing Efficiency Score | ACN | Revenue per kWh delivered |

Post-pricing utilization is **simulated using price elasticity (−0.3)** rather than compared against historical data — because comparing against historical data only measures model accuracy, not pricing impact. This is standard econometric practice when a live A/B test is unavailable.

## How to Run

### 1. Install dependencies

```bash
pip install pandas numpy matplotlib seaborn scikit-learn xgboost shap openpyxl
```

### 2. Place raw datasets

Put all raw files into `rawdatasets/` as listed in the structure above.

### 3. Run preprocessing

```bash
python data_preprocessing.py
```

Generates `final_datasets/acn_processed.csv` and `final_datasets/aligned_urbanev_data.csv`.

### 4. Run EDA

```bash
python eda.py
```

Saves all charts to `plots/`.

### 5. Run the agents

```bash
python run_agents.py
```

Trains the demand model on full UrbanEV data, then runs 15 episodes of the agentic pricing loop. Outputs saved to `outputs/`.

### Fresh start (reset learning)

```bash
del memory\*.json
del memory\*.pkl
```

Run this if you want Q-values and models to reset completely.

## Key Design Decisions

**Why epsilon-greedy bandit?**
The pricing agent needs to balance exploiting known good tariffs (85%) with exploring new ones (15%). A bandit is lighter than full RL and interpretable — Q-values directly show which tariff worked best for each utilization level.

**Why constrained tariff selection?**
Without constraints, the bandit can pick ₹8 for a congested station. Surge zones (util > 0.80) are restricted to ₹15–₹24; discount zones (util < 0.30) to ₹8–₹15. This encodes domain knowledge while still allowing the bandit to learn within each range.

**Why domain-knowledge Q-value initialization?**
Starting all Q-values at 0 causes early random selection bias — ₹8 gets selected first by chance, gets reward 0.2, and dominates for hundreds of episodes. Initializing with sensible priors (high tariffs favored for high-util bins) gives the bandit a correct starting point.

**Why elasticity simulation for utilization change?**
Comparing predicted_util vs historical UrbanEV util only measures demand model accuracy — not whether pricing worked. Elasticity simulation (util_after = util_before × (1 + −0.3 × price_change%/100)) directly answers: "if prices changed by X%, how much did demand shift?"

**Why two datasets?**
ACN provides real session-level revenue data (actual kWh, actual durations) for grounding revenue metrics. UrbanEV provides city-scale temporal demand patterns for training the prediction model. Neither alone is sufficient.

## Results Interpretation

After 15 episodes, the learned Q-values should show:

| Utilization Bin | Expected Best Tariff |
|---|---|
| very_high (>0.80) | ₹21–₹24 (surge) |
| high (0.60–0.80) | ₹18–₹21 |
| medium (0.40–0.60) | ₹15 (standard) |
| low (0.20–0.40) | ₹12 |
| very_low (<0.20) | ₹10–₹12 (mild discount) |

Zone-specific utilization breakdown (printed each episode):
- **Surge zones**: util_after < util_before → congestion relieved 
- **Discount zones**: util_after > util_before → empty stations filling 
- **Standard zones**: util_after ≈ util_before → stable
